# Polish Detail Workflow States

Status: ready-for-agent
Owner: unassigned
Related: `public/index.html`, `.scratch/detail-page-generation/PRD.md`

## Problem

The current detail-page workflow exists, but the feeling of step-by-step generation is still weak.

## Goal

Make the detail-page mode feel more like a guided e-commerce workflow and less like a generic form that instantly returns a mock result.

## Scope

- clarify stage transitions
- improve button text, status text, and result messaging
- support the feeling of:
  - upload product
  - choose style
  - confirm first screen
  - generate full detail-page set

## Acceptance Criteria

- the workflow stages are easy to understand at a glance
- users can tell what step they are in
- button and status language matches the detail-page workflow
- front-end remains stable in the local preview

## Notes

This task is still front-end focused and should help define backend behavior later.
