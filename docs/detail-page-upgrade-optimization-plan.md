# 详情页生成优化与升级实施文档

## 文档目的

这份文档不是替代原来的 7 步主流程文档，而是作为当前项目进入 Step 7 之后的专项升级文档。

它用来约束后续优化顺序，避免同时改太多问题导致系统再次失稳。

这份文档解决的是两类事情：

1. 当前已经暴露出来的问题，按优先级重新整理
2. 后续明确要增加的能力，按阶段逐步接入

核心原则不变：

- 一次只推进一个阶段
- 每个阶段做完后必须先验收
- 当前阶段没通过，不能进入下一阶段

---

## 当前总判断

项目现在不是单点 bug，而是四类问题叠在一起：

1. 生成质量问题
2. 前端接线和状态问题
3. 流程产品形态问题
4. 模型能力与系统流程耦合问题

这意味着不能继续零散修补，而要重新按阶段推进。

---

## 当前问题总表

### A. 生成质量问题

1. 首图生成对视觉风格的遵循不够强
2. 这个问题不止高奢风，所有视觉风格都需要强化
3. 首图容易退回成“白底图轻微处理”，而不是风格化电商首图
4. 款式/颜色模块容易退化成“原图 + 叠字”
5. 各模块对参考图结构的遵循还不够稳定
6. 智能优化提示词目前更像固定模板，不是真正基于用户输入和参考图动态优化

### B. 前端接线与交互问题

1. 场景使用图模块还不能正式生成
2. 需要确保所有模块都可以正式生成
3. `AI 一键生成`、`保存这项`、`导出摘要` 等按钮有未接通问题
4. 不同区域按钮颜色不统一，主动作按钮应统一为紫色体系
5. `一键整理详情页` 需要改名为 `统一导出详情页`
6. `统一导出详情页` 不应强制要求每个模块都先生成完成
7. 文字处理模块布局过窄，影响整体舒适度
8. 视觉风格选择项过少，需要支持用户自定义

### C. 状态管理与系统稳定性问题

1. 详情页生成和场景图生成切换时，生成结果区文案会串状态
2. 页面内多个模式的结果区、状态文案、按钮状态没有完全隔离
3. 如果后续支持并发生图，当前状态管理结构会更容易出错

### D. 产品升级需求

1. 增加“老板模式”或“一键生成整套详情页”
2. 用户不需要逐步确认，也能直接跑完整套详情页
3. 自动跑完整套之后，仍然允许对单模块提意见和返工
4. 这套快捷模式不能破坏原来的分步模式

### E. 架构层问题

1. 当前模型 API 和流程逻辑仍然耦合偏紧
2. 后续换模型时，必须保证流程层不跟着崩
3. 需要把模型调用、提示词构建、工作流编排、任务状态管理进一步分层

---

## 升级总目标

这轮升级的目标不是简单“把 bug 修完”，而是把系统推进到下面这个状态：

1. 所有主模块都能稳定正式生成
2. 所有视觉风格都能真正影响生成结果
3. 各模块更严格遵循参考图结构
4. 所有关键按钮都真实可用
5. 前端状态切换不串内容
6. 用户可以选择“分步模式”或“老板模式”
7. 模型以后可以替换，而流程不需要大改

---

## 执行总规则

### 规则 1：先修断路，再修质量，再做升级

先处理“功能根本不可用”的问题，再处理“生成质量不够好”的问题，最后再做新增能力。

### 规则 2：一个阶段只解决一类主问题

每一阶段必须有明确边界，不能一边修接线，一边改老板模式，一边重构模型层。

### 规则 3：每一阶段必须先验收

只有当前阶段验收通过，才进入下一阶段。

### 规则 4：不为追求快而破坏当前主流程

当前已经成立的主流程仍然是：

1. 摘要整理
2. 摘要确认
3. 首图生成
4. 首图确认
5. 模块规划
6. 参考图归类
7. 单模块生成
8. 整页整理

快捷模式必须建立在这套流程之上，而不是粗暴绕开。

---

## 分阶段实施计划

## 第 1 阶段：修通关键断路点

### 本阶段目标

先确保“该能点的能点、该能生成的能生成、该能导出的能导出”。

### 本阶段范围

1. 接通场景使用图正式生成
2. 补齐所有模块正式生成入口
3. 接通 `AI 一键生成`
4. 接通 `保存这项`
5. 接通 `导出摘要`
6. 把关键主动作按钮统一成紫色风格
7. 把 `一键整理详情页` 改为 `统一导出详情页`
8. `统一导出详情页` 不要求所有模块都先生成完成

### 本阶段不做什么

1. 不做老板模式
2. 不做模型层重构
3. 不深改生成质量

### 验收标准

1. 所有主模块都有正式生成入口
2. 场景图模块可以真实发起生成
3. `AI 一键生成` 能回填内容
4. `保存这项` 能触发真实保存和状态刷新
5. `导出摘要` 点击后有明确结果
6. `统一导出详情页` 在部分模块未生成时也能进入

---

## 第 2 阶段：修复状态管理和前端布局稳定性

### 本阶段目标

先把前端系统状态修稳，避免模式切换、模块切换时出现串内容和错乱。

### 本阶段范围

1. 修复场景图生成和详情页生成之间的结果区串状态问题
2. 拆分不同模式的结果显示状态
3. 拆分不同模式的按钮 loading 状态
4. 优化文字处理模块布局，让它铺满模块宽度
5. 统一模块卡片间距、模块对齐和移动端视觉密度

### 本阶段不做什么

1. 不做视觉风格强化
2. 不做老板模式
3. 不做模型层替换

### 验收标准

1. 详情页模式和场景图模式来回切换，结果区不串文案
2. 文字处理模块不再挤成窄侧栏
3. 模块卡片视觉密度更统一
4. 主要操作按钮的 loading 和禁用状态不串

---

## 第 3 阶段：重做视觉风格约束

### 本阶段目标

让“视觉风格”真正成为生成的强约束，而不是轻提示。

### 本阶段范围

1. 强化所有视觉风格，不只高奢风
2. 让风格在首图生成中更明显生效
3. 让风格在后续模块生成中也能延续
4. 增加自定义视觉风格输入
5. 支持“预设风格 + 用户自定义补充”的混合输入

### 重点说明

这一阶段解决的不是“某一个风格不好”，而是整个系统对风格输入的执行力不足。

### 本阶段验收标准

1. 同一商品切换不同风格，结果应有可见差异
2. 首图不再只是白底图轻微处理
3. 模块图也能延续已选风格，而不是只受原图影响
4. 用户自定义风格输入能够参与生成

---

## 第 4 阶段：重做模块生成质量

### 本阶段目标

让每个模块真正像“模块”，而不是原图叠字或弱改图。

### 本阶段范围

1. 修复款式/颜色模块退化成叠字的问题
2. 加强所有模块对参考图结构的继承
3. 强化模块视觉重点的执行力
4. 强化模块文案角度和文字排版执行力
5. 减少重复图、九宫格退化、海报化退化

### 本阶段验收标准

1. variants 模块不再只是叠英文或叠标签
2. details / scenes / params / trust 等模块都更像真实电商详情页板块
3. 模块输出更能看出“参考图定版式、首图定基调”
4. 系统自己写出的视觉重点，正式生成结果能真正体现

---

## 第 5 阶段：重做智能优化提示词系统

### 本阶段目标

把“智能优化提示词”从固定模板改造成真正动态的提示词优化能力。

### 本阶段范围

1. 让优化逻辑读取用户输入
2. 让优化逻辑读取参考图方向
3. 让优化逻辑区分场景图模式和详情页模式
4. 让优化结果不是固定句式套壳
5. 支持用户继续修改优化结果

### 本阶段验收标准

1. 不同商品、不同参考图、不同输入下，优化结果应明显不同
2. 优化结果必须和当前任务模式相关
3. 优化结果不能再是固定模板换词

---

## 第 6 阶段：增加“老板模式 / 一键整套详情页”

### 本阶段目标

增加一套面向不想一步一步操作用户的快捷模式。

### 本阶段范围

1. 在详情页生成入口增加 `一键生成整套详情页`
2. 自动完成：
   - 摘要整理
   - 首图生成
   - 模块规划
   - 单模块生成
   - 整页整理
3. 生成结束后支持：
   - 整体通过
   - 单模块返工
   - 单模块提意见重做

### 本阶段边界

这不是删除原流程，而是在原流程旁边增加一条快捷路径。

### 本阶段验收标准

1. 用户不做逐步确认，也能生成一套可看的详情页
2. 快捷模式完成后，仍能进入单模块返工
3. 不会破坏现有分步模式

---

## 第 7 阶段：模型解耦与可替换架构

### 本阶段目标

让未来换模型时，系统流程不需要大改。

### 本阶段范围

1. 拆分 workflow layer
2. 拆分 prompt builder layer
3. 拆分 model provider adapter layer
4. 拆分 task state layer
5. 统一模型返回结构
6. 统一错误处理和重试机制

### 本阶段验收标准

1. 切换模型时不需要改主流程页面逻辑
2. 不同模型返回能被统一适配
3. 生图流程、摘要流程、识图流程不会因为换模型而整体失稳

---

## 当前推荐推进顺序

按现在的项目状态，推荐顺序如下：

1. 第 1 阶段：修通关键断路点
2. 第 2 阶段：修复状态管理和前端布局稳定性
3. 第 3 阶段：重做视觉风格约束
4. 第 4 阶段：重做模块生成质量
5. 第 5 阶段：重做智能优化提示词系统
6. 第 6 阶段：增加老板模式
7. 第 7 阶段：模型解耦与可替换架构

---

## 当前阶段建议

如果按“先确保每一步无误后再下一步”的原则，当前最合理的下一步是：

`先进入第 1 阶段：修通关键断路点`

原因是：

1. 现在还有按钮没接通、模块没接通的问题
2. 如果基础交互都没通，后面谈生成质量会被假象干扰
3. 先把动作链路打通，后面每一轮验收才有意义

---

## 当前工作约束

从现在开始，后续优化必须遵守下面这条约束：

`必须先完成当前阶段，并由用户验收通过后，才能进入下一阶段。`

这条规则优先级高于“想一次性把所有问题都修掉”的冲动。
