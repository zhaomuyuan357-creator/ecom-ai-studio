**产品名称：** Ecom AI Studio（电商AI工作室）

## 一、产品概述

### 1.1 一句话定义
一款面向电商卖家的AI内容工具——上传白底商品图，一键生成场景图和商品详情页。

### 1.2 核心价值主张
| 维度 | 传统方式 | 本产品 |
|------|---------|--------|
| 单张成本 | 20-80元（外包拍摄） | ≈0.1元（API调用） |
| 时间 | 数小时-数天 | 数秒-数十秒 |
| 技能要求 | 会PS / 找设计师 | 会打字就行 |
| 差异化 | 模板千篇一律 | AI根据描述生成独特场景 |

### 1.3 商业模式
- **一次性部署费：** 299-999元（标准版/专业版/旗舰版）
- **API费用：** 商户自备API密钥，自行承担调用费用
- **核心价值：** 卖的是"工具 + Prompt + 工作流"，不是API能力本身
- **类比：** 卖电饭煲，不卖电。商户自己买电（API费用），你卖做饭的工具。

### 1.4 开发与生产模型策略
| 阶段 | 模型 | 用途 | API费用 |
|------|------|------|---------|
| **开发测试** | 通义千问（qwen-image-2.0 + qwen3.6-plus） | 跑通流程、验证功能 | 开发者承担 |
| **生产上线** | ChatGPT image-2（通过中转站） | 商户实际使用 | 商户承担 |

> 通义千问仅用于开发阶段快速验证。上线前需解决GPT image-2中转站问题（价格≤0.2元/张）。

### 3.2 功能一：AI场景图生成

**用户流程：**
```
① 打开软件 → ② 输入API密钥（首次） → ③ 上传白底商品图 → ④ 输入场景描述（或选模板） → ⑤ 点击生成 → ⑥ 查看/下载结果
```

| 功能 | 描述 | 优先级 |
|------|------|--------|
| API密钥配置 | 用户在设置中输入API密钥，保存在本地加密存储 | P0 |
| 图片上传 | 支持拖拽或点击上传白底商品图（JPG/PNG，≤10MB） | P0 |

| 场景模板 | 预设8个常用场景，点击自动填入描述（可修改） | P0 |
| AI生成 | 调用AI API，将白底图+描述融合生成场景图 | P0 |
| 结果展示 | 显示生成的图片，支持放大查看 | P0 |
| 下载保存 | 支持下载生成的图片到本地 | P0 |
| 生成历史 | 本地保存最近的生成记录（图片+描述） | P1 |

### 3.3 功能二：AI详情页生成

**用户流程：**
```
① 上传白底商品图 → ② 粘贴竞品详情页URL（或上传参考截图） → ③ 填写商品信息（引导问题+自由文本） → ④ AI分析参考页面结构 → ⑤ 先生成首图确认风格 → ⑥ 满意后批量生成剩余图片 → ⑦ 逐张查看 → ⑧ 对某张不满意 → 打字描述问题 → AI重新生成该张
```

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 白底图上传 | 上传商品白底图 | P0 |
| 参考URL输入 | 粘贴竞品详情页链接，系统尝试抓取 | P0 |
| 参考截图上传 | URL抓取失败时，提示用户上传截图 | P0 |
| 商品信息输入 | 3-4个引导问题 + 自由文本补充框 | P0 |
| 详情页结构分析 | AI分析参考页面的布局结构（图片数量、内容顺序、排版方式） | P0 |
| 首图确认 | 先生成第1张（首图），用户确认风格方向 | P0 |
| 批量生成 | 首图确认后，批量生成剩余图片，显示进度 | P0 |
| 单张修改 | 用户对某张图打字描述问题，AI重新生成该张 | P0 |
| AI自动文案 | AI根据商品信息自动生成详情页文案（标题、卖点、参数） | P0 |
| 整套下载 | 一键下载所有详情页图片（按顺序命名） | P1 |
| 尺寸选择 | 默认790px宽，支持手动选择其他尺寸 | P1 |

**商品信息输入方式（混合引导）：**
```
1. 产品名称？→ [输入框]
2. 最大卖点？（最多3个）→ [输入框]
3. 主要使用场景？→ [输入框]
4. 其他补充？→ [自由文本框]
```

**详情页学习方式：学习内容框架，视觉风格AI自由发挥**
- 分析竞品详情页的：图片数量、内容顺序、每张讲什么
- 不复制竞品的视觉风格，由AI在选定的风格模板内自由生成
- 用户可选择预设视觉风格（高端奢华/简约现代/活力缤纷等）

**URL抓取失败处理：**
```
粘贴URL → 尝试抓取 → 成功 → 分析
                     → 失败 → 提示"无法抓取，请上传截图" → 用户上传截图 → 分析截图
```

## 四、技术架构

### 4.1 整体架构

**MVP阶段：网页应用（Express + 静态HTML）**
**后续阶段：改造为Electron桌面应用**

```
┌─────────────────────────────────────────────┐
│  Express 代理服务器（本地运行）                │
│  ┌────────────────────────────────────────┐ │
│  │  server.js                             │ │
│  │  - 提供静态文件（public/index.html）    │ │
│  │  - 代理API请求（解决CORS问题）          │ │
│  │  - 路由：                               │ │
│  │    POST /api/generate-scene            │ │
│  │    POST /api/analyze-page              │ │
│  │    POST /api/generate-detail           │ │
│  └───────────────┬────────────────────────┘ │
└──────────────────┼──────────────────────────┘
                   │ HTTPS
                   ▼
          ┌──────────────────┐
          │  阿里云百炼        │
          │  (qwen-image-2.0) │
          │  (qwen3.6-plus)   │
          └──────────────────┘

浏览器访问 http://localhost:3000
┌──────────────────────────────────────────────┐
│  public/index.html                           │
│  - 暗黑主题UI（现有HTML改造）                  │
│  - 场景图生成功能                              │
│  - 详情页生成功能                              │
│  - API密钥配置                                │
│  - 生成历史记录                               │
└──────────────────────────────────────────────┘
```

### 4.2 技术选型

| 层级 | 方案 | 说明 |
|------|------|------|
| **后端** | Express.js | 代理API请求，解决CORS，提供静态文件 |
| **前端** | 原生HTML + CSS + JS | 基于现有HTML改造，不引入React（MVP保持简单） |
| **API调用** | 阿里云百炼（OpenAI兼容格式） | 开发测试用 |
| **密钥存储** | localStorage（MVP） | 后续用electron-store加密 |
| **本地存储** | localStorage + 服务端JSON | 存储生成历史和配置 |

### 4.3 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| MVP用网页还是Electron？ | **网页** | 快速验证，后续再改造Electron |
| 是否需要后端？ | **需要Express代理** | 浏览器直接调API有CORS限制 |
| 密钥存在哪里？ | **服务端环境变量（开发）/ localStorage（生产）** | MVP阶段简化 |
| Prompt存在哪里？ | **服务端代码中** | 前端不可见 |
| 安装包如何分发？ | **微信发送** | MVP阶段最简单 |

### 4.4 API调用流程

**场景图生成：**
```
前端：上传白底图 + 场景描述
  ↓
后端：组装prompt + 编码图片为base64
  ↓
调用 qwen-image-2.0 API
  ↓
返回生成的图片（base64/URL）
  ↓
前端：展示结果 + 缓存到本地
```

**详情页生成：**
```
前端：上传白底图 + 竞品URL/截图 + 商品信息
  ↓
后端 Step1：调用 qwen3.6-plus 分析竞品页面结构
  ↓
后端 Step2：基于分析结果 + 商品信息，逐张调用 qwen-image-2.0 生成
  ↓
每张图生成完成后实时返回前端（SSE/轮询）
  ↓
前端：显示进度 + 展示结果
  ↓
用户修改某张 → 打字描述 → 重新生成该张
```

### 4.5 安全设计

| 风险 | 措施 |
|------|------|
| API密钥泄露 | 开发阶段Key在服务端环境变量中，前端不接触 |
| Prompt泄露 | Prompt在服务端组装，前端不可见 |
| 用户密钥安全 | 生产阶段用户Key存localStorage，后续升级为加密存储 |
| 网络传输安全 | 所有API调用走HTTPS |
